# 개인 지식 온톨로지 구축기 — Part 1: 옵시디언 Vault 셋업

> Notion에 흩어진 자료를 옵시디언 기반 온톨로지로 체계화하는 과정의 첫 번째 기록

## 왜 온톨로지인가

기존의 파일/폴더 정리와 온톨로지의 차이:
- 폴더는 **트리 구조** — 하나의 문서가 하나의 폴더에만 속함
- 온톨로지는 **그래프 구조** — 하나의 문서가 여러 맥락에 동시 연결
- "세션"이 헷갈릴 때 → 관련된 업무 노트, 학습 자료, 프로젝트 기록을 한번에 조회 가능

## 도구 선택

- **옵시디언(Obsidian)**: 마크다운 기반, 로컬 저장, 무료, [[양방향 링크]] 지원
- **Notion에서 이전하는 이유**: 관계 기반 탐색, 로컬 데이터 소유권, 플러그인 생태계

## 설계 과정

### 접근법 결정

3가지 접근법을 비교 검토:
1. 폴더 중심 (Folder-First) — 직관적이나 온톨로지보다 "정리된 폴더"에 가까움
2. **노트 타입 중심 (Type-First)** ← 채택 — 폴더는 노트 종류별, 관계는 속성+링크로
3. 플랫 구조 + MOC — 유연하나 초보자에게 혼란

### Vault 구조

```
vault/
├── 0-Inbox/      # 미분류 임시 보관
├── 1-Concept/    # 개념 사전 (1개념 = 1노트)
├── 2-Project/    # 업무/프로젝트 기록
├── 3-Resource/   # 독서, 강의, 참고자료
├── 4-Log/        # 일상, 여행, 회고
├── 5-Output/     # 블로그 초안, 결과물
├── Templates/    # 자동 적용 템플릿
└── MOC/          # 주제별 허브 문서
```

**핵심 원칙**: 폴더 = 노트의 종류, 속성/태그 = 노트의 맥락, [[링크]] = 노트 간 관계

### 속성 체계

모든 노트에 공통 속성 7개:
- `type`, `category`, `tags` — 분류
- `learning-need`, `learning-status` — 학습 상태 추적
- `visibility` — 블로그 공개 가능 여부
- `created`, `updated` — 시간 추적

Learning을 별도 카테고리가 아닌 **속성으로 관리**한 이유:
→ 어떤 카테고리의 노트든 "학습 필요"를 표시할 수 있어야 하기 때문

### 태그 vs 링크 vs 속성

| 도구 | 역할 | 예시 |
|---|---|---|
| #태그 | 주제 분류 (넓은 범위) | `#marketing/analytics` |
| [[링크]] | 구체적 관계 연결 | `[[GA4]]` |
| 속성 | 구조화된 메타데이터 | `learning-status: in-progress` |

## 셋업 과정

### 1. Claude Code로 파일 구조 일괄 생성

폴더 구조, 템플릿 6개, MOC 4개, 샘플 노트 3개를 Claude Code로 한번에 생성.
옵시디언을 열기도 전에 구조가 준비됨.

### 2. 옵시디언 설치 및 Vault 열기

- obsidian.md에서 다운로드
- 생성된 vault/ 폴더를 Vault로 열기

### 3. 플러그인 설치 (4개)

| 플러그인 | 역할 |
|---|---|
| Dataview | 속성 기반 쿼리 (온톨로지 엔진) |
| Templater | 폴더별 자동 템플릿 |
| Calendar | 날짜 기반 탐색 |
| Obsidian Importer | Notion 변환 (마이그레이션 시) |

### 4. Templater 설정

각 폴더에 전용 템플릿 매핑:
- 1-Concept → concept-template.md
- 2-Project → project-template.md
- (이하 동일 패턴)

→ 폴더에서 새 노트를 만들면 속성이 자동으로 채워짐

### 5. 검증

- Dashboard.md 열기 → Dataview 테이블에 샘플 노트 표시 확인
- Graph View → 노트 간 연결선 확인
- 새 노트 생성 → 템플릿 자동 적용 확인

## 사용된 도구와 기술

- **Claude Code**: brainstorming 스킬로 설계, Plan 모드로 구현 계획, 파일 일괄 생성
- **옵시디언**: Vault, 커뮤니티 플러그인, Properties, Graph View
- **Dataview 쿼리**: SQL과 유사한 문법으로 노트를 필터링/정렬
