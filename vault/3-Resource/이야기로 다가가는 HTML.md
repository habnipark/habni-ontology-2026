> **p.32 FTP 방식을 바꿔준 http프로토콜**
> - R 데이터를 다루는 형식에 따라 그에 걸맞는 접속 규칙을 활용했던 과거의 불편함을 바꿔준 혁신이 ‘http 프로토콜’방식이다. http에 대한 재밌는 사항은, http 뒤에 붙는 두개의 슬래시는 사실 팀 버너스리가 “아무 이유 없이 붙였다” 라고 주장한 바가 있고, 그는 “포워드 슬래시는 사실 불필요하다. 당시에는 그게 좋을줄 알았다”라고 아쉬움을 토로했다.
> - A  웹 환경 발달 관련해서 과거의 역사와 사례를 잘 찾아보면 좋을 것 같다. FTP 방식은 무엇인지도 자세히 알아보자.

> **p.49 HTML의 헤드 요소**
> - R 헤드 요소를 교과서적으로 정의하면 ‘문서의 메타데이터’의 집합이다. 이 안에는 문서의 제목을 다루는 <title>, 메타 정보를 담는<meta>, 태그 스타일 시트를 다루는<css>, 자바스크립트를 다루는<script>로 구성되어있다.

> **p.49 HTML의 <meta>태그**
> - R 메타 정보를 담는<meta>태그는 다른 데이터를 설명해 주는 역할을 한다. 즉 일정한 규칙에 따라 효율적으로 콘텐츠에 부여하는 정보이다. 메타데이터는 해당 페이지를 이해하고 더 자세한 정보를 파악하는 데에 도움이 되는 데이터이다.
